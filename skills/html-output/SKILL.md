这个 Skill 的完整规则将在后续版本中设计。

## 方向

- 在适合展示复杂信息、结构化内容、审阅结果、确认项或轻量交互的场景下，优先输出 HTML 而不是纯 Markdown。
- 输出的 HTML 应该信息密度高、界面简洁、美观、阅读体验好，并尽量保持单文件、自包含。
- 如果需要用户补充信息，HTML 可以包含表单、按钮、输入框等交互元素。
- 用户补充信息的后端提交和后续 AI 读取能力是附加能力，不是这个 Skill 的唯一用途。

## 第一版范围

第一版不规定：

1. HTML 必须长什么样。
2. 页面必须包含哪些表单字段。
3. 用户补充信息必须是什么格式。
4. AI 必须如何生成提交逻辑。
5. AI 必须如何组织页面结构。
6. 每个 HTML 都必须提交到后端。

这些内容留到后续真实使用后再设计。
