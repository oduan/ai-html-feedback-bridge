这个 Skill 的完整规则将在后续版本中设计。

## 方向

- 在适合展示复杂信息、表单或交互入口的场景下，优先输出 HTML 而不是纯 Markdown。
- 输出的 HTML 可以包含表单、按钮、输入框等交互元素，用于收集用户的补充信息。
- 收集到的用户补充信息可以通过后端 API 保存，后续可由 AI 读取和处理。

## 第一版范围

第一版不规定：

1. HTML 必须长什么样。
2. 页面必须包含哪些表单字段。
3. 用户补充信息必须是什么格式。
4. AI 必须如何生成提交逻辑。
5. AI 必须如何组织页面结构。

这些内容留到后续真实使用后再设计。
